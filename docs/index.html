Driftlock Choir

Precision timing infrastructure for distributed systems. Chronometric interferometry delivers picosecond-scale τ recovery and sub-ppb Δf estimation across GHz RF bands.

  • ✅ 13.5 ps clean recovery at 2.4 GHz
  • ✅ 0.09 ps residual error at 5.8 GHz
  • ⚠️ Multipath calibration pending (ns-scale drift)

Quick Start

Install

pip install -r requirements.txt

Run CLI

python run_experiment.py   --band 2.4GHz   --channel-profile line_of_sight   --duration-ms 2.0   --sampling-rate-msps 40   --tau-ps 13.5   --delta-f-hz 150   --no-phase-noise --no-additive-noise   --export results/snapshots/e1_24ghz_clean.json

Key Figures

System-level schematic

Two-way RF exchange, beat-note mixing, and phase-slope estimation combine to recover τ and Δf with picosecond precision.

  • • Mix oscillators at 2.4 GHz and 5.8 GHz
  • • Apply fractional delay channels to model τ
  • • Phase unwrapping + regression yields τ and Δf

Limitations & Roadmap

Current limitations

  • RF multipath produces large τ bias without calibration.
  • Hardware captures awaiting RTL-SDR lab integration.
  • Uncertainty dashboards require new Monte Carlo ingestion.

Active roadmap

  • Design calibration sequences for multipath compensation.
  • Automate CLI validation in CI (clean, multipath, hardware dry run).
  • Publish updated uncertainty dashboards for GHz regimes.
